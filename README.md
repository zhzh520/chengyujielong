# 成语接龙智能对话系统

一个基于本地大模型 **Gemma3:4b** 与自定义成语库实现的成语接龙 Web 应用，采用前后端一体化架构，浏览器直接交互，支持对话历史记录、优先成语库匹配、AI 智能兜底。

## 功能特性

 **本地成语库优先匹配**：从 `chengyuku.txt` 读取成语，保证接龙规范准确

**Gemma3:4b 大模型兜底**：成语库无匹配时自动调用本地 Ollama 模型

**浏览器可视化界面**：美化气泡对话样式，支持滚动历史

**完整交互逻辑**：输入成语、AI 接龙、历史记录、一键清空

**单文件运行**：前后端一体化，无需拆分部署

**回车快捷发送**：输入后按 Enter 即可发送，操作流畅

## 实验步骤

### 步骤 1：准备 Ollama 环境

1. 确保电脑已安装 **Ollama**

2. 以管理员身份打开终端，拉取模型：

   ```
   ollama pull gemma3:4b
   ```

3. 等待模型下载完成，Ollama 会自动在后台启动服务。

4. Ollama 默认接口地址：`http://localhost:11434`，无需额外配置。

### 步骤 2：准备成语库文件

1. 在项目目录下新建 `chengyuku.txt`
2. 按**一行一个成语**格式填入四字成语
3. 确保路径与代码中读取路径一致

## 实验原理

### 1. 成语库优先匹配逻辑

1. 程序启动时读取 `chengyuku.txt`。
2. 逐行加载，过滤非四字内容，存入集合。
3. 用户输入成语后，取最后一个汉字。
4. 遍历成语库，寻找首字匹配的成语。
5. 找到则直接返回，保证规则严格、不出错。

### 2. 大模型兜底逻辑

1. 成语库无匹配时触发 AI 调用。
2. 构造提示词：要求用指定字开头、只返回四字成语。
3. 通过 POST 请求访问 Ollama 本地接口。
4. 获取模型返回结果，过滤多余内容。
5. 返回前端展示。

### 3. 前后端交互逻辑

- 前端：HTML 渲染界面，JS 发送请求、接收数据、渲染气泡。
- 后端：Flask 提供 3 个接口
  - `/`：返回网页
  - `/play`：接收用户成语，返回 AI 回复与历史
  - `/history`：获取历史记录
  - `/clear`：清空历史

### 4. 异常情况处理

- 输入非四字内容 → 提示 “请输入四字成语”
- Ollama 未启动 → AI 显示 “接不上啦”
- 成语库无匹配 → 自动调用大模型补充

### 5.可能出现的问题及解决方法

1. **无法连接 Ollama**

- 原因：Ollama 服务未启动
- 解决：重新打开终端运行 `ollama run gemma3:4b`

2. **成语库加载失败**

- 原因：文件名错误 / 路径错误 / 编码不是 UTF-8
- 解决：检查文件名、路径，重新保存为 UTF-8 格式

**3. 端口 5000 被占用**

- 解决：修改代码中 `app.run(port=5000)` 为 5001/8000 等
